## 说明

- 导出 XML 图谱的文件需使用 VS Code QTZ-HIS 插件的"导出工程"功能。
- v9.1 来源为 Git 仓库；其余版本来源于原医生站 `svn\tags`。
- 项目使用时若没有对应版本，可采取就高原则，例如：v8.3 使用 v8.4 的图谱。
- 若需其他版本或专科系统的代码图谱，可提 issue 或 PR 补充。
- 导出代码前建议先在本地目录执行 `git init` 初始化仓库；导出完成后统一提交一次作为基线。后续每次修改代码前，先通过插件拉取最新代码并提交到本地仓库，保留完整变更历史，再开始修改。
- 图谱未覆盖的文件可自行补充到本地 XML 图谱中，但请勿提交到 agents 仓库。
