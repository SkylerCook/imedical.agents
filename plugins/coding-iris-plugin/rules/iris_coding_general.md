# 通用编辑安全规则

本文保存跨语言、跨模块通用的编辑约束。前端、后端和部署任务还必须读取各自专项规则。

## 基本原则

- 修改前先读取目标工程 profile、相关源码和调用链，理解现有结构和命名习惯。
- 优先沿用目标工程已有模式，不为单次改动引入新框架、新目录或新抽象。
- 保持改动范围最小，只修改与任务直接相关的文件和代码块。
- 不回滚、不覆盖用户已有改动；遇到同文件已有变更时先读懂再合并。
- 不做无关重构、无关格式化、无关元数据更新。

## 文件编辑安全

- 保留原文件编码、换行风格和特殊 EOF。
- 历史 CSP/JS/CSS/ObjectScript 文件避免整文件重写，优先局部补丁。
- 生成或移动文件时遵循目标工程 profile 的目录与命名约定。
- 本地缺少远程已有实现时，可用 MCP 只读读取远程内容补上下文，再按需落本地做增量修改。
- 删除文件、批量覆盖、大范围同步、数据库变更等高风险操作必须先取得用户明确确认。

## 质量约束

- 一个函数或方法只承担一个清晰职责。
- 重复逻辑达到真实维护成本时再抽取公共函数。
- 错误处理集中、清晰，避免在多个调用点复制复杂兜底判断。
- 新增 TODO 只能记录明确后续演进方向，当前代码必须保持可运行。
- 命名使用业务语义，避免含糊缩写。

## 完成检查

- 检查改动范围，确认没有无关换行、编码、空白或格式化变更。
- 后端变更如需验证，用户明确要求后再使用目标工程 MCP 编译相关类。
- 前端 CSP/JS/CSS 变更如需部署，用户明确要求后再按工作流转换编码、上传并验证。
- 无法运行必要验证时，在回复中说明原因和剩余风险。
