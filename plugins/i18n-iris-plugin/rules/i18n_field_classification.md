# 国际化数据分类规则

执行本规则前必须先读取 `.agents/config/i18n_project_profile.md`。本规则用于在链路定位完成后，对每个用户可见文本进行分类，确定处理方式，为编码改造阶段提供结构化输入。

## 适用范围

适用于页面级和打印级国际化需求中，用户可见文本的分类和处理方式映射。链路定位完成后使用本规则。

## 分类体系

每个用户可见文本必须归入以下五类之一：

### 固定文案

**判断标准**：代码中硬编码的标题、页码、标签、金额单位、括号标识、状态提示等。

**处理方式**：
- 页面级：使用页面级翻译 helper（如 `%Trans`、`%Translate`）。
- 打印级：优先使用整句占位符翻译，避免只翻译词根后继续拼接中文语序。

**翻译表类型**：`frontend-extra`（前端）或 `backend-message`（后端）。

### 字典展示值

**判断标准**：从 Global、SQL、持久类字段、字典表字段直接取出的原文展示值。包括科室、性别、医院、用户、医护人员、医嘱项等。

**处理方式**：
- 使用字典/表字段翻译 helper（如 `GetTransXxx`）。
- 翻译位置应贴近原始字段来源，不在最终变量上无脑套 `GetTransXxx`。
- 首次遇到新字典时，优先补公共门面方法。

**翻译表类型**：`backend-dict`。

**贴近原则**：当代码明确从某个数据源取出展示值时，在该取值后立即调用翻译。若后续变量又被外部接口返回值覆盖，则不应对覆盖后的最终变量继续套同一个翻译方法。这能避免把不同来源、不同业务含义的文本误认为同一个字典字段。

### 业务输入

**判断标准**：备注、用户手工录入说明、病人或医生临时输入内容。

**处理方式**：默认不翻译。

**翻译表类型**：不进入翻译表。

### 外部接口返回

**判断标准**：当前工程无法看到源码或归属其他业务组的接口返回值。

**处理方式**：当前工程不二次翻译，转交对应业务组。

**翻译表类型**：不进入翻译表。在输出中标记"外部接口返回，转交 xx 业务组"。

### 未确认来源

**判断标准**：调用链能看到方法名但无法确定底层数据来源。

**处理方式**：继续追链路；追不到时标记待确认，不猜测改造。

**翻译表类型**：暂不进入翻译表。在输出中标记"待确认"。

## 页面级 vs 打印级差异

| 维度 | 页面级 | 打印级 |
|---|---|---|
| 固定文案来源 | DOM 文本、JS 消息、后端提示 | 打印数据中的标签、标题、分页、金额单位 |
| 字典展示值来源 | 后端返回前端的表字段值 | 打印数据中的人员、科室、医院、性别、医嘱项 |
| 额外禁止翻译 | id/code/类名/URL 等 | 同上 + 模板变量名、JSON 字段名、条码值 |
| 结构化字典描述 | 不单独处理 | `上级-下级` 格式需确认分隔符保留范式 |

## 分类结果输出格式

分类完成后输出字段分类清单，供编码改造阶段消费：

```markdown
# 字段分类清单 - {需求号}
| # | 文本 | 位置 | 类型 | 处理方式 | 备注 |
|---|------|------|------|----------|------|
| 1 | 床号 | 后端打印数据 | 固定文案 | 页面级翻译helper，整句占位符 | |
| 2 | 内科-内分泌科 | 后端打印数据 | 字典展示值 | GetTransLoc，分隔符保留 | |
| 3 | 备注内容 | 后端打印数据 | 业务输入 | 不翻译 | 用户手工录入 |
| 4 | 检验结果值 | 外部接口返回 | 外部接口 | 不翻译，转交xx业务组 | |
| 5 | 某字段值 | 后端类方法 | 未确认 | 待确认 | 调用链不明 |
```

## 禁止翻译清单（通用）

以下内容在任何分类中都不翻译：

- `id`、`code`、状态码、字典编码。
- 类名、表名、字段名、方法名、URL、接口参数名。
- 用于保存、查询、关联的持久化标识。
- 模板变量名、JSON 字段名、条码值、业务编码。
