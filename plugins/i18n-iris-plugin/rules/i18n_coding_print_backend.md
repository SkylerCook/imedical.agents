# 后端打印国际化编码规则

执行本规则前必须先读取 `.agents/config/i18n_project_profile.md` 和 `i18n_coding_backend.md`。本规则适用于后端生成实际打印返回数据、XML 打印模板名、条码数据、申请单/告知单/病理单等打印数据的多语言改造。

链路定位前必须先读取 `i18n_link_tracing.md`，按实际代码和实际返回数据确认链路，不预设 `PrintTemp`、`templateId`、`GetXMLTemplateId` 或 JSON 形态。

## 打印模板选择

- 后端取得 XML 打印模板代码后，必须通过项目 profile 指定的国际化模板匹配 helper 选择语言模板。
- 当前 IRIS 医生站默认公共方法为 `##class(DHCDoc.Util.Translate).GetI18nXMLPrintTemplate(xptCode, patientId, sessionStr)`；目标工程 profile 有覆盖时，以 profile 为准。
- 国际化模板 helper 必须以原模板代码作为默认返回值；未配置目标语言模板、当前语言为源语言、语言目录缺失时，都要回退原模板。
- 业务代码禁止把模板匹配结果无保护地覆盖为空。若项目 helper 不能保证 fallback，调用侧必须显式兜底原模板。
- 模板命名约定、语言代码大小写、语言目录来源以 project profile 为准，不得硬编码某个环境的语言 id。

## 打印数据文案

打印数据中的用户可见文本，先按 `i18n_field_classification.md` 分类再改造：

- 固定文案（标题、分页、金额单位、括号标识、急/补/科研等）：使用页面级翻译 helper。带数字或业务值的打印文本优先使用整句占位符翻译，例如页码、床号、费用、申请单标题，避免只翻译词根后继续拼接中文语序。
- 字典展示值（人员、科室、医院、性别、医嘱项等）：使用字典/表字段翻译 helper；不得混用页面级翻译 helper。翻译位置应贴近原始字段来源。
- 业务输入（备注、用户手工录入）：默认不翻译。
- 外部接口返回：当前工程不二次翻译，转交对应业务组。
- 只翻译展示值，不翻译模板变量名、JSON 字段名、条码值、业务编码、状态编码、URL、类名、表名或字段名。

## 结构化字典描述

- 对形如 `上级-下级` 的字典描述，先确认目标工程的字典翻译范式。
- 若项目约定目标翻译保留同构分隔符和层级顺序，例如 `内科-内分泌科 -> nk-nfmk`，允许先按完整字典描述翻译，再按同一分隔符截取展示。
- 若目标翻译不保证保留分隔符，禁止翻译后再用 `$p`、`$e`、`contains` 等方式解析目标语言文本；应改用结构化字段或把派生展示文案作为页面级翻译处理。
- 翻译种子和翻译质量检查必须记录此类字段的分隔符保留要求。

## 完成检查

- 扫描本次改造文件，确认没有新增 `b //`、`b ;`、临时调试输出或测试断点。
- 扫描打印链路中的裸中文拼接，确认用户可见文本已按固定文案或字典展示值分流处理。
- 中文源语言、缺少目标语言模板、存在目标语言模板三种场景都应验证模板代码 fallback 行为。
