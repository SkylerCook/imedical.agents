# HIS/医疗系统目标语言翻译质量规则

执行会生成目标语言翻译的任务前，必须先读取 `.agents/config/i18n_project_profile.md`，确认源语言、目标语言、语言代码、翻译表目录和项目术语约定。

本规则适用于中文 HIS/医疗系统 UI 文案翻译到任意目标语言。法语只是目标语言专项约束之一。

## 理解源文案

- 先理解中文源文案的业务语境，不逐字硬翻。
- 区分按钮、菜单、表头、提示、错误信息、确认语、业务状态、字典展示值和医疗术语。
- 不额外添加源文案没有的业务含义。
- 不翻译业务编码、状态编码、字段名、类名、方法名、URL、接口参数或持久化标识。

## 医疗与 HIS 术语

- 医疗术语优先使用目标语言中常见、正式、行业可接受的表达。
- HIS 场景下保持界面操作语义清晰，例如保存、提交、审核、撤销、打印、导出、申请、医嘱、诊断、材料、费用等。
- 同一中文源文案在同一目标语言和同一业务上下文中尽量保持同译。
- 若项目 profile 或术语表已有译法，优先沿用项目译法。

## 目标语言正字法

- 翻译必须符合目标语言的拼写、大小写、空格、标点和 UI 表达习惯。
- 法语必须正确使用重音符和变音符，例如 `é`、`à`、`è`、`ù`、`â`、`ê`、`î`、`ô`、`û`、`ë`、`ï`、`ü`、`ç`。
- 不得为了编码方便去掉目标语言必要的重音符、变音符或自然标点。
- 目标语言需要自然撇号时，允许使用 `'` 或 `’`，例如法语 `l'utilisateur`、`d'admission`。

## UI 长度控制

- 在不损失语义和医学严谨性的前提下，译文应尽量简洁，适合界面显示。
- 按钮、菜单、表头、页签优先使用短译。
- 提示、错误信息、确认语允许略长，但必须清晰可读。
- 不硬性要求译文不超过中文长度；法语等语言通常会更长，但应避免无意义扩写。
- 对 XML 打印模板、窄列头、按钮等固定宽度位置，应优先选择不明显溢出的表达。

## 占位符与结构保持

- 必须保留 `{0}`、`{1}`、`%1`、`${name}` 等占位符，不得增删。
- 可按目标语言语序调整占位符顺序，但必须保持占位符含义准确。
- 不改 HTML、XML、JS、SQL、ObjectScript 结构，不翻译标签名、属性名、变量名或语法关键字。
- 翻译包含 HTML 文本时，只翻译用户可见文本，不破坏标签和属性。

## 结构化输出安全

- 翻译文本层面允许自然语言需要的 `'` 和 `’`。
- 生成代码、SQL、ObjectScript 种子、JSON、XML、HTML 属性等结构化输出时，必须按目标格式正确转义。
- 不得输出会破坏字符串、SQL、XML、JSON、HTML 属性或 ObjectScript 语法的未转义内容。
- 不能为了避免转义而牺牲目标语言正字法或自然表达。

## 翻译表要求

- `HIS系统最佳目标语言翻译` 列按本规则生成。
- 没有把握的医疗术语、缩写或上下文，翻译表备注中标记 `需确认`。
- 对同一源文案出现多种可能译法时，按业务上下文选择最适合 UI 的译法，并在备注说明差异。
