## 判断规则

按顺序判定，命中即停止：

1. 用户明确说“后续按需导出文件”“需求处理中再导出”“刚新建”“空工程”“只有零散文件”“需求处理工作区”时，使用 `intent-first-on-demand-export`。
2. 本文件已有明确 `contextMode` 时，沿用该值；除非用户明确更新，否则不要改。
3. 缺少可识别构建/运行入口，且业务代码文件很少或分散时，使用 `intent-first-on-demand-export`。
4. 只有在本地同时具备较完整源码目录、构建或运行配置、项目文档或用户确认“本地就是主要工程”时，才使用 `codebase-complete`。
5. 如果无法判断，默认使用 `intent-first-on-demand-export`。

## 按需导出约束

- 本地少量或零散文件只代表当前已导出的需求上下文。
- 不得围绕单个文件生成完整 `AGENTS.md` 架构说明。
- 不得基于零散文件推断整体架构、主模块边界或完整调用链。
- 本地已有文件最多列为“当前已导出/已存在文件”，不得写成系统组成、核心模块或完整调用链。
- 每个需求先确认目标页面、类、JS、CSP 或业务对象，再导出相关文件处理。
