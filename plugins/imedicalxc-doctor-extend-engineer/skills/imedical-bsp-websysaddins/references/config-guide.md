# 插件管理配置说明

## 配置入口

进入 iMedical 系统的 **【开发工具管理】→【插件管理】**界面进行中间件配置维护。

> 配置界面分为两个区域：
> - **上方表头区**：维护中间件的基本信息（程序集、路径、版本等）
> - **下方列表区**：维护中间件包含的方法及各方法的详细配置

---

## 一、表头区配置项详解

### 1. 控件代码

- **作用**：中间件的唯一标识编码，用于系统内部识别
- **格式**：字符串标识符
- **示例**：`JavaReadDevice`
- **注意**：创建后一般不可修改，需保持唯一性

### 2. 程序集名

- **作用**：指定 Java 程序的主入口类的完整限定名（全限定类名）
- **格式**：包名 + 类名（含 main 方法的类）
- **示例**：`ReadDevice`
- **注意**：对于 Java 类型中间件，此处填写主类的全限定名；与下方列表区的「程序集」字段对应

### 3. 调用 ID 名

- **作用**：定义 JS 中使用的全局对象名，即类的一个实例标识
- **格式**：合法的 JavaScript 标识符
- **示例**：`JavaReadDevice`
- **使用方式**：在 CSP 页面的 `<ADDINS>` 标签引入后，即可通过此名称调用方法

### 4. 应用文件路径（DLL 文件路径）

- **作用**：指定中间件组件的文件路径（Windows 客户端）
- **格式**：相对于 `web/addins/plugin` 目录的相对路径
- **规则**：
  - 支持  `.zip` 压缩包
  - 如果是 **zip 包**，客户端下载后会自动解压并按内部描述注册
  - 多个文件用逗号分隔，**逗号第一位为主程序入口**，后续为依赖文件
  - **逗号第二位不用写入子目录路径**
- **示例**：`JavaReadDevice/JavaReadDevice.zip`

### 5. 版本号（DLL 包版本）

- **作用**：标识当前中间件包的版本号，控制客户端自动更新
- **机制**：当客户端安装的版本与此处配置的版本不一致时，自动从服务端下载安装最新版本
- **格式**：语义化版本号，如 `1.0.0`、`1.0.1`
- **建议**：每次更新发布时递增版本号

### 6. 功能说明

- **作用**：描述该中间件的功能用途，便于管理和辨识
- **示例**：`Java版本读卡`

### 7. 是否可见 / 是否激活

| 配置项 | 说明 |
|--------|------|
| **是否可见** | 控制该中间件是否在客户端界面中可见显示 |
| **是否激活** | 控制该中间件是否生效可用（✅ 勾选后生效） |

### 8. Linux 下的应用文件路径

- **作用**：指定 Linux 环境下中间件文件的部署路径
- **格式**：绝对路径或相对于插件目录的路径
- **说明**：当 iMedical 客户端在 Linux 系统上时使用此项

### 9. 客户端 IP

- **作用**：限制该中间件只对特定 IP 范围的客户端生效
- **格式**：IP 地址段 / CIDR 格式，多种组合用逗号分隔
- **示例**：`192.168.[1-18].[20-], 192.168.[2,4]*, 10.*.10.*, ...`
- **说明**：留空表示不限制，所有客户端均可使用

---

## 二、列表区配置项详解（方法级别,JAR包可忽略）

### 关键字段说明

| 配置项 | 说明 |
|--------|------|
| **方法名** | 对象包含的方法名，JS 要调用需要在这里配置上，JAR包的话可忽略，方法名固定为cmd |
| **调用清除** | 调用方法前清对象数据（会清除属性值，对象带有属性不要勾选）， |
| **即时调用** | 立即向中间件发出请求调用方法 |

---

## 三、JS 调用链路

配置完成后，在 CSP 页面中的典型调用链路：

```
<ADDINS 标签引入>
    ↓
调用ID名（全局对象实例）
    ↓
方法名（具体执行的方法，JAR包时固定为cmd）
```

**完整调用示例**：
```javascript
// 在 CSP 页面中通过 ADDINS 引入后
JavaReadDevice.cmd(params);
// ↑调用ID名    ↑方法名
```

## 五、版本更新流程

1. 开发完成新版本 Java 工程
2. 通过 Maven 打包生成新的 `.jar` / `.zip` 文件
3. 将文件部署到服务端 `web/addins/plugin/` 目录下对应路径
4. 在【客户端动态库维护】中更新**版本号**
5. 客户端下次启动或刷新时自动检测版本差异并下载更新

## 六、常见问题排查

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 客户端未自动更新 | 版本号未递增 | 修改版本号为更高值并保存 |
| JS 调用找不到对象 | 调用ID名配置错误 | 检查 CSP 中 ADDINS 引用的调用ID名是否与配置一致 |
| 方法调用无响应 | 是否激活未勾选 | 在列表区确认对应方法的「是否激活」为 Y |
| 特定客户端无法使用 | 客户端IP 限制了范围 | 检查表头区和列表区的客户端IP配置是否包含目标IP |
| Linux 环境无法加载 | 未配置 Linux 路径 | 补充「Linux 下的应用文件路径」和「Linux 下库文件」配置 |
