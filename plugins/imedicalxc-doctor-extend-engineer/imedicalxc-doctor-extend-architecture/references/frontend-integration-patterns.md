# 前端集成调用模式

## 典型调用场景

| 业务场景 | 调用时机 | 第三方系统 | 调用方式 | 是否阻塞 |
|---------|---------|-----------|---------|---------|
| **医嘱提交前库存校验** | 点击提交按钮时 | SPD 系统 | 实时 HTTP 调用 | 是，需等待返回结果 |
| **诊断录入时医保校验** | 输入诊断编码后 | 医保系统 | 实时 HTTP 调用 | 是，需显示校验结果 |
| **开具处方时合理用药审查** | 保存处方前 | 合理用药系统 | 实时 HTTP 调用 | 是，需展示审查结果 |
| **患者身份核验** | 接诊时 | 公安身份系统 | 实时 HTTP/中间件 | 是，需验证身份 |
| **打印处方时电子签章** | 点击打印按钮时 | CA 签章系统 | 中间件/DLL | 是，需等待签章完成 |
| **卡号读取** | 刷卡时 | 读卡器设备 | 中间件/OCX | 否，异步回调 |

## 调用模式分类

### 模式 1：同步阻塞调用

**适用场景**：需要立即获得结果才能继续业务流程。

```
用户操作 → 调用外部接口 → 等待响应 → 根据结果决定是否继续
```

**关键要点**：
- 必须显示 loading 提示，避免用户重复操作
- 根据返回结果决定是否继续业务流程
- 必须处理超时和错误情况

### 模式 2：异步回调调用

**适用场景**：不需要立即结果，支持后台处理。

```
用户操作 → 调用外部接口 → 立即返回 → 后台处理 → 完成后回调通知
```

**关键要点**：
- 立即返回任务 ID，不阻塞用户操作
- 后台轮询查询任务状态
- 完成后通过回调或消息通知用户

### 模式 3：中间件封装调用

**适用场景**：第三方提供 DLL/OCX/ActiveX 等非浏览器兼容组件。

**技术通道**：
```
前端 JS → 医为中间件 API → 本地 DLL/OCX → 第三方系统
```

**前置判断：第三方流程意图**

中间件本身是透明通道，技术选型前必须先分析第三方设计的业务流程属于哪种形式：

#### 形式 A：通知型（Fire-and-Forget）

第三方接收数据后自行处理交互（弹窗、语音、打印），HIS 不等待返回值。

- HIS 侧：调用后不做结果判断，不阻塞业务流程
- 第三方责任：自己处理用户提示、操作结果展示
- 超时：5-10 秒，超时仅记录日志，不阻断流程
- 接口特征：返回 void 或简单状态码
- 示例：第三方语音叫号、第三方打印机驱动、写卡器通知

#### 形式 B：交互型（Wait-for-Response）

第三方不做 UI 交互，只返回处理数据，HIS 必须等待返回值并基于返回值设计交互。

- HIS 侧：必须等待返回值，根据返回值控制业务流程和界面
- HIS 侧责任：根据返回值设计提醒/弹窗/确认界面，与操作用户交互
- 第三方责任：只返回处理结果数据，不做 UI 交互
- 超时：30 秒，超时须阻断流程并明确提示用户
- 接口特征：返回结构化数据（成功/失败/数据内容）
- 示例：CA 电子签章（返回签章结果供 HIS 展示）、合理用药审查（返回审查结论）
- 特殊：部分厂商还需 HIS 将用户操作结果回传给第三方（双向交互）

**关键要点**：
- **先分析流程意图，再选择技术通道** — 不能用“中间件”一词掩盖流程差异
- 检查中间件是否已加载
- 通知型：失败/超时只记录日志，不中断业务流程
- 交互型：失败/超时必须阻断业务流程并提示用户
- 根据流程形式设置合理的超时时间（通知型短，交互型长）

## 错误处理策略

**超时处理**：设置超时定时器，防止接口长时间无响应阻塞用户操作。

## 安全规范

**防重放攻击**：生成唯一请求 ID 并缓存，防止重复请求。

## 性能优化

**接口合并**：批量调用外部接口，减少网络请求。

**本地缓存**：对外部接口结果进行本地缓存，减少重复调用。

## 调试与监控

**接口调用日志**：统一封装接口调用日志，记录调用参数、响应结果和耗时。

**性能监控**：监控接口响应时间，对慢接口进行告警。

## 最佳实践总结

1. **始终使用外部接口层**：所有第三方接口调用必须通过 `/comoe/interface/{Vendor}/{Module}.js`
2. **处理超时和错误**：外部接口不可靠，必须有完善的错误处理和用户提示
3. **阻塞场景需明确提示**：同步调用时必须显示 loading 或进度提示
4. **敏感数据加密**：患者隐私数据必须加密传输
5. **支持降级**：关键业务要有备用方案，避免单点故障
6. **记录调用日志**：便于问题排查和性能优化
7. **版本兼容性**：考虑第三方接口版本升级时的兼容处理
8. **限流保护**：防止高频调用导致第三方系统过载
