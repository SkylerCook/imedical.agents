## 严重程度判定规则

| 严重程度 | 判定条件 | 是否记录产品组 |
|----------|----------|----------------|
| **高** | 存在 ERROR 日志、N+1 性能问题、接口超时>3s | 必须记录 |
| **中** | 存在 WARN 但功能正常、单次耗时偏高 | 记录 |
| **低** | 无异常、无性能问题 | 不记录 |

## 问题归属规则

### 目录归属
- `{产品组}` 按**问题涉及的产品组**划分（非接口所属模块）
- 涉及多个产品组时，按**问题最严重**的归类
- 严重程度低的不创建目录

### Feign 调用问题归属
- Feign 提供方接口本身无问题（单次响应正常耗时）→ **不列为问题方**
- 重复调用、N+1 逐条调用 → 归到**调用方**
- 提供方缺少批量接口需配合新增 → 列为"**需配合新增批量接口**"
- 提供方内部代码缺陷（如慢SQL、逻辑错误）→ 归到**提供方**

### 报告内"涉及产品组"表格
- 必须列出所有需要修改的产品组
- 每个产品组标注：模块代码、模块名称、角色（核心缺陷方/调用方/需配合新增批量接口）、问题性质
- 无问题的 Feign 提供方不列入
