# 报告模板

## 输出触发条件

仅当用户明确说"输出文件"、"生成MD"、"生成文档"、"写入文档"、"保存报告"、"导出报告"、"输出报告"时使用此模板。

## 输出目录

报告默认输出到当前项目目录，用户可另行指定。路径格式：
```
{输出目录}\{问题产品组}\{模块}_{方法名}性能分析报告.md
```

- `{问题产品组}` 按分析出的问题涉及的产品组划分，非接口所属模块
- 涉及多个产品组时按问题最严重的归类
- 产品组和模块映射参见 [[application-mapping.md]]
- 严重程度低的不创建目录

## 严重程度判定

| 严重程度 | 判定条件 | 是否创建目录 |
|----------|----------|-------------|
| **高** | ERROR 日志、N+1、超时>3s | 必须创建 |
| **中** | WARN 但功能正常、耗时偏高 | 创建 |
| **低** | 无异常、无性能问题 | 不创建 |

## 报告结构

```markdown
# `{方法名}` 接口性能分析报告

## 基本信息
| 项目 | 值 |
|------|-----|
| 产品组 | {产品组名称} |
| 模块 | {模块代码} - {模块名称} |
| 接口路径 | ... |
| 控制器 | ... |
| 业务实现 | ... |
| TraceId | ... |
| 入参 | ... |
| 环境 | ... |
| 请求时间 | ... |
| 总耗时 | ... |
| 日志量 | ... |
| 严重程度 | 高/中/低 |

## 调用链路
（树形文本图，标注 Feign/DB/N+1 等关键操作，标注精确的文件名:行号）

## 耗时分解（瀑布图）
（时间轴风格，分阶段展示，每阶段标注占比和关键操作）

## 操作次数统计
（表格：操作类型、次数、表/接口）

## 接口耗时排名（Top 10）
| 排名 | 操作 | 单次耗时 | 调用次数 | 估算总耗时 | 占比 |
|------|------|----------|----------|------------|------|

## 核心问题分析
（每个问题一个小节：代码位置精确到文件名:行号、问题代码片段、影响分析、Graylog 证据）

## 优化方案
（每个方案标注 优先级+预期收益，含优化前后代码对比）

## 总结
（汇总表：方案、预期收益、实施位置、难度、优先级 + 根本原因 + 执行建议）
```

## 质量标准

- 每个数值必须有来源（日志提取 > 代码推算 > 合理估算，标注来源）
- 代码位置精确到 `文件名:行号`，N+1 问题必须标注循环入口行和循环内调用行
- 优化方案必须给出可执行的代码示例（优化前/后对比）
- 优先级排序以"投入产出比"为准（收益大 + 改动小 = 最高优先级）
- 保留已生成报告，仅新增不覆盖
