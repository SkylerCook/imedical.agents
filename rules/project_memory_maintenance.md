# 项目记忆维护规则

## 用途

`.agents/memory/project_memory.md` 用于帮助后续 agent 快速续接上下文，不是正式项目文档、规则手册或完整 changelog。

目标是让新会话在 2 分钟内理解：

- 当前工程主线。
- 稳定工程事实。
- 已做出的长期决策。
- 容易重复踩坑的经验。
- 仍然有效的后续建议。

## 固定结构

项目记忆建议保持以下结构：

1. `当前状态`
2. `工程事实`
3. `重要决策`
4. `关键入口`
5. `长期经验`
6. `待办建议`
7. `最近变化`

可以按需增删小节，但不要变成长篇流水账。

## 可以写入

- 长期有效事实：工程类型、业务主线、模块边界、关键路径。
- 架构或流程决策：提交范围、翻译存储默认、MCP 事实来源。
- 跨任务坑点：会影响后续多次任务的陷阱、验证方式、已确认限制。
- 已验证的重要经验：能避免回归或错误部署的经验。
- 仍有效待办：未来任务开始时应优先知道的事项。
- 最近变化摘要：只写对后续任务有影响的变化。

## 不要写入

- 完整规则全文。
- 翻译表格式、长示例、完整 SQL 或大段代码。
- 长 changelog 或每一步操作流水账。
- 临时命令输出、一次性排查过程、即时状态噪声。
- 可从代码、rules、profile、templates 直接查到的重复内容。
- 已完成且对后续没有影响的事项。

## 更新原则

- 阶段性完成后再更新；不要每做一个小动作就追加。
- 优先替换、合并、压缩旧内容，不无限追加。
- 每次更新顶部 `更新时间` 和 `最近变化`。
- 如果新增内容属于 i18n 规则、项目适配或迁移步骤，优先更新对应文件：
  - i18n 规则：`.agents/rules/i18n_*.md`
  - 项目适配：`.agents/config/i18n_project_profile.md`
  - 迁移说明：`.agents/plugins/i18n-iris-plugin/templates/`
  - 入口约束：`AGENTS.md`
- memory 只保留这些文件的入口链接和长期有效摘要。

## 审查清单

更新 memory 后检查：

- 是否能在 2 分钟内读完主要上下文。
- 是否删除了可由其它规则文件承载的细节。
- 是否保留了关键决策和坑点。
- 是否没有记录敏感连接信息。
- 是否没有把临时过程当成长期记忆。
