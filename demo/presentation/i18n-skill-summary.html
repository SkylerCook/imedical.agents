i18n-coding Skill 指导意义总结

需求 #6096272：挂号打印小条多语言改造

Skill 版本：i18n-iris-plugin

Skill 核心指导原则

1

触发条件明确

当任务要求国际化编码改造时自动触发。编码改造只保留源语言文案 key，不绑定具体目标语言，源语言和默认目标语言以项目 profile 为准。

2

必读规则体系

要求读取 10 项规则：项目配置、规则索引、前后端规则、字典翻译门面、打印后端规则、链路定位、字段分类、验证规则等，形成完整的知识体系。

3

阶段化执行

复杂需求推荐 5 阶段执行：链路定位 → 字段分类 → 编码改造 → 模板/种子 → 验证。每个阶段有明确的输入输出和产出物。

4

经验沉淀机制

需求完成后检查可复用经验，更新 docs/demand-com-exp.md。避免重复踩坑，积累项目级知识。

本次需求执行过程

阶段化执行路径

1

链路定位

按照 i18n_link_tracing.md 规则，从入口 CSP 出发，追踪调用链：

opadm.regist.csp → regist.js:PrintOut → $.cm(OPRegistBroker) → GetOPPrintData → 返回 JSON 打印数据

输出：链路事实报告，确认渲染路径为 XML 打印模板 + Lodop 打印控件。

2

字段分类

按照 i18n_field_classification.md 规则，对用户可见文本分类：

类型字段示例处理方式
固定文案"元"、"就诊号:"、"第"、"号"后端 %Trans() / 前端 $g()
字典展示值科室、医生、医院、职称、时段GetTransXxx() 字典翻译门面
业务输入患者姓名、备注不翻译
3

编码改造

后端改造：

  • 7 处固定文案用 %Trans() 包裹
  • 7 处字典字段调用 GetTransXxx() 翻译
  • 新增 3 个翻译方法：GetTransSessionType、GetTransTimeRange、GetTransClinicGroup
  • 新增 PrintTemplateCode 字段实现模板国际化

前端改造：

  • 费用标签用 $g() 包裹
  • PersonPay 改用 parseFloat() 提取数值
  • 模板代码从后端动态获取，而非硬编码
4

验证与经验沉淀

按照 i18n_verify.md 进行验证，发现子方法 GetRegitems 中的 ARCIMDesc 遗漏翻译。

新增经验 6.5：字典翻译检查需覆盖被调用子方法。主方法中的字典字段已翻译，但被调用的子方法中的字典字段可能遗漏。
